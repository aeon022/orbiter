# @a83/orbiter-cli

CLI for [Orbiter CMS](https://orbiter.sh) тАФ scaffold projects, manage users, export content, and sync media with git.

[![npm](https://img.shields.io/npm/v/@a83/orbiter-cli?color=8b7cf8)](https://www.npmjs.com/package/@a83/orbiter-cli)
[![License: MIT](https://img.shields.io/badge/license-MIT-green.svg)](https://github.com/aeon022/orbiter/blob/main/LICENSE)

---

## Install

```bash
npm install -g @a83/orbiter-cli
```

Requires **Node.js 20+**.

---

## Commands

### `orbiter init [dir]`

Scaffold a new Astro + Orbiter project.

```bash
orbiter init my-site
cd my-site
npm install
npm run dev
```

Creates:
- `astro.config.mjs` тАФ pre-configured with `@a83/orbiter-integration`
- `content.pod` тАФ fresh pod with a default Posts collection and admin user
- `src/pages/` тАФ example index and blog pages using `orbiter:collections`
- `package.json`, `.gitignore`, GitHub Actions workflow

---

### `orbiter add-user`

Add an admin or editor user to an existing pod.

```bash
orbiter add-user --pod ./content.pod
```

Interactive prompts: username, password, role (`admin` or `editor`).

```bash
# Non-interactive
orbiter add-user --pod ./content.pod --username editor1 --role editor
```

---

### `orbiter export`

Export all published entries to JSON or Markdown files.

```bash
# JSON (default)
orbiter export --pod ./content.pod --out ./export

# Markdown тАФ one .md file per entry, frontmatter for all non-body fields
orbiter export --pod ./content.pod --out ./export --format md

# Single collection
orbiter export --pod ./content.pod --out ./export --collection posts
```

Output structure:

```
export/
тФЬтФАтФА posts/
тФВ   тФЬтФАтФА my-first-post.json
тФВ   тФФтФАтФА another-post.json
тФФтФАтФА pages/
    тФФтФАтФА about.json
```

### `orbiter unpack`

Extract media BLOBs from the pod to regular files. Use before `git commit` when hosting on platforms with ephemeral filesystems (Netlify, Vercel, GitHub Pages).

```bash
orbiter unpack --pod ./content.pod --media-dir ./media
```

After unpacking, BLOBs are removed from `_media.data` and a `storage.mode=git` flag is set in `_meta`. The pod stays small for git тАФ media files live alongside it.

---

### `orbiter pack`

Restore media BLOBs from files back into the pod. Run at build time before `astro build`.

```bash
orbiter pack --pod ./content.pod --media-dir ./media
```

Reads each file in `./media`, inserts it as a BLOB, and clears the `storage.mode` flag.

## Git sync workflow

For static hosting (Netlify, Vercel, GitHub Pages):

```
1. Edit content in the Orbiter admin
2. orbiter unpack  тЖТ  extracts media to ./media/
3. git add content.pod media/ && git commit && git push
4. CI: orbiter pack тЖТ restores BLOBs тЖТ astro build тЖТ deploy
```

### GitHub Actions

```yaml
# .github/workflows/build.yml
name: Build & Deploy

on:
  push:
    branches: [main]
  workflow_dispatch:    # тЖР triggered by Orbiter's build webhook

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: '20' }
      - run: npm install
      - run: npx orbiter pack --pod ./content.pod --media-dir ./media
      - run: npx astro build
      - uses: actions/deploy-pages@v4
```

> A ready-made workflow file is generated by `orbiter init` and downloadable from **Settings тЖТ GitHub тЖТ тмЗ Workflow** in the admin.
