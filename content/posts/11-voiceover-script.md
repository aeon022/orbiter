# Orbiter Demo — Voiceover Script

Ready to paste into ElevenLabs or read aloud. Each block = one scene. Pause 2 seconds between blocks.

---

## Scene 1 — Hook (15 sec)

What if your CMS was a single file? No database server. No cloud account. No Docker. Just one SQLite file called a POD — and a desktop app to manage it. This is Orbiter.

## Scene 2 — Dashboard (20 sec)

The dashboard gives you the full picture. This calendar widget shows what's scheduled, what's published, and what's expiring — all at a glance. Below that, your recently edited entries. And the dock at the bottom gives quick access to everything — hover over a collection to preview its latest entries.

## Scene 3 — Calendar (30 sec)

The calendar page is the full view. Every entry shows up as a color-coded chip. Blue for scheduled. Gold for expiring. Green for published. Click a day to see what's there. Filter by status. Arrow keys to navigate between months. T to jump back to today. Click any entry to open it directly in the editor.

## Scene 4 — Editor (25 sec)

The editor has a block-based content area on the left and schema fields on the right. This table field is a mini-spreadsheet — add rows, add columns, tab between cells. It's stored as a simple JSON array in the entry data. And down here in the meta panel, you can see the scheduled publish date.

## Scene 5 — Cross-Pod (30 sec)

Cross-pod copy. Export your entire pod as JSON — every collection, every entry, in one file. Then import it into another pod. Collections are created automatically if they don't exist yet. Choose skip or overwrite for duplicate entries. Staging to production in two clicks.

## Scene 6 — The POD (20 sec)

This is it. One file. Your entire CMS. It's a standard SQLite database — you can open it with any database tool. Copy it. Email it. Put it in Dropbox. No migrations, no export wizards. Just a file.

## Scene 7 — Closing (20 sec)

Orbiter is MIT-licensed, self-hosted, and works on any shared hosting via FTP. Sixteen field types, a block editor, scheduled publishing, multilingual support, a media library, and a desktop app with auto-update. Check it out on GitHub — link in the description.
