**Neu: Kalenderansicht, Cross-Pod-Copy und Desktop Auto-Update für Orbiter CMS**

Orbiter ist ein Headless-CMS für Astro, das alles in einer einzigen `.pod`-Datei speichert — Content, Medien, User, Settings. Kein Cloud-Konto, kein Docker, kein Abo. Diese Woche sind drei neue Features live:

📅 **Kalenderansicht**
Eine eigene Kalenderseite mit Monatsraster. Jeder Eintrag erscheint farbcodiert — blau für geplant, gold für ablaufend, grün für veröffentlicht. Klick auf einen Tag zeigt die Einträge in einer Sidebar. Das Dashboard hat jetzt auch ein Mini-Kalender-Widget.

🔄 **Cross-Pod Import/Export**
Pod-Dateien sind portabel — jetzt kann man einen ganzen Pod als JSON exportieren und in einen anderen importieren. Collections werden automatisch erstellt, Einträge können übersprungen oder überschrieben werden. Perfekt für Staging → Produktion.

🖥️ **Desktop App v0.2.0**
Auto-Update über GitHub Releases, Universal macOS Binary (M1 + Intel in einer DMG) und Ein-Klick-Backup.

Dazu: ein neues Tabellen-Feld — Mini-Spreadsheet direkt in der Entry-Sidebar.

Orbiter ist MIT-lizenziert, braucht keine Cloud und deployt auf jedes Shared Hosting via FTP. Gebaut für kleine Firmen und Freelancer, die ein CMS ohne Infrastruktur-Overhead wollen.
